# _BOOTSTRAP_IA_IMAGEM — kit de partida para o repositório novo

Esta pasta é um **kit de partida**. O conteúdo dela deve ser **copiado para um
repositório NOVO e separado** (o projeto da IA de imagem especializada na
coleção). Ela vive aqui no repo dos livros só como área de preparação — não faz
parte da produção dos livros.

## Como usar (autor)

1. Crie um repositório novo e privado no GitHub (ex.: `orquestrador-narrativa-ia`).
2. Copie **todo o conteúdo desta pasta** para a raiz do repo novo.
3. Abra a sessão do Claude Code nesse repo novo **já com o Fable 5** e diga algo
   como: *"Leia o CLAUDE.md e siga o protocolo de início."*
4. O agente vai ler tudo, avaliar as próprias capacidades, criticar/atualizar o
   plano e propor o arranque da Fase 1 antes de gerar qualquer custo.

## O que tem aqui

| Arquivo | Para quê |
|---|---|
| `CLAUDE.md` | Regras do projeto novo + protocolo de início para o agente |
| `UNIVERSO.md` | Briefing completo do universo (61 personagens, estilo, regras visuais) |
| `PLANO.md` | Passo a passo (Fase 1 enxuta → Fase 4), para o agente avaliar e melhorar |
| `INFRAESTRUTURA.md` | O que o autor já tem e pode reaproveitar (criar só 1 conta nova) |
| `referencia_visual/` | DNA visual: guias de padronização/cores/cenários + as 6 imagens que existiam no repo-mãe |

## Importante

- **As imagens de treino reais (caps 1–5 do Livro 1) NÃO estão aqui** — estão na
  máquina do autor. As 6 imagens em `referencia_visual/imagens/` são só amostras de
  referência (pranchas + infográficos). O autor sobe o dataset completo na Fase 1.
- A **fonte canônica** do universo continua no repo dos livros
  (`00_FUNDACAO_E_GUIAS/`). Os documentos aqui são um resumo autossuficiente; em
  divergência, a fonte canônica prevalece.
